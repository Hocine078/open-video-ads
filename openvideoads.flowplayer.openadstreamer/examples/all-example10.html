Example 10 - A mixed RTMP and HTTP playlist

This example illustrates how to serve up a mixture of HTTP and RTMP streams (shows and ads). Zone '20' is a zone that serves all types of ads (RTMP and HTTP). This example plays a pre-roll ad (could be either HTTP or RTMP) and then a show stream (RTMP) followed by another pre-roll ad of some type and then a HTTP version of the show stream (each limited to 10 seconds).


The configuration for this example is: 


flowplayer("a.example", "../dist/flowplayer-3.1.5.swf", {
    plugins: {
        rtmp: {
            url: "../dist/flowplayer.rtmp-3.1.3.swf"
        },
        rtmpInstream: {
            url: "../dist/flowplayer.rtmp-3.1.3.swf"
        },
        controls: {
            autoHide: "always"
        },

        openAdStreamer: {
            url: '../dist/OpenAdStreamer.swf',

            "shows": {
                "streams": [
                   {
                      "file":"rtmp://ne7c0nwbit.rtmphost.com/videoplayer/mp4:the-black-hole.mp4",
                      "duration":"00:00:10"
                   },
                   {
                      "file":"http://streaming.openvideoads.org/shows/the-black-hole.mp4",
                      "duration":"00:00:10"
                   }
                ]
            },

            "ads": {
                "servers": [
                    {
                       "type": "OpenX",
                       "apiAddress": "http://openx.openvideoads.org/openx/www/delivery/fc.php"
                    }
                ],
                "schedule": [
                   {
                      "zone": "15",
                      "position": "pre-roll"
                   }
                ]
            },

            "debug": {
                "levels": "fatal",
                "debugger": "firebug"
            }
        }
    }
});
